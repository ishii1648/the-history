## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
ユーザー報告: 地図上の表示（国名・都市名・河川名等のラベル）に日本語と英語が混ざって表示されている。TASK-40（情報パネル・タイムラインの羊皮紙風デザイン化）を実装した辺りから発生している現象とのこと。ただし TASK-40 の変更は app.css のみ（PR #52 で確認済み）であり、地図ラベルの文字列自体（NAME → 日本語のフラットマップ、name-ja.json 適用箇所）を直接変更していないため、真因は別タスクにある可能性が高い。TASK-40 前後でマージされた TASK-27（都市マーカー）・TASK-33（解説パネル）・TASK-45（agent-loop エスカレーション基準文書）等、同時期の変更を洗い出し、日本語/英語混在の実際の発生箇所（国名・都市名・河川名・HRE 領邦名のどれか、特定の年代か全年代か）を特定すること。考えられる原因の切り分け観点: (1) name-ja.json に未登録の名前が存在し仕様どおり英語フォールバックしているだけ（既存仕様であり不具合ではない可能性）(2) 直近の機能追加（都市・解説パネル等）で新規追加された名前が name-ja.json への追加を漏らしている (3) 表示ロジック側の参照ミス（ja マップを渡し忘れている等）。再現手順は実機（ビルド + dev サーバ + ブラウザ）で年代を切り替えながら確認して確立すること。
